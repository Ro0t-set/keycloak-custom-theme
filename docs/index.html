KC Custom Theme

Keycloak Custom Theme

A clean, modern theme for Keycloak authentication pages with custom styling and branding support.

Get Started View on GitHub

Installation

Deploy the custom theme using Docker. The theme is automatically loaded when you copy it to the themes directory.

Dockerfile 
FROM quay.io/keycloak/keycloak:26.1.5

COPY ./custom/ /opt/keycloak/themes/custom/

ENTRYPOINT ["/opt/keycloak/bin/kc.sh"]

Build and run

docker build -t keycloak-custom .

docker run -p 8080:8080 \
  -e KEYCLOAK_ADMIN=admin \
  -e KEYCLOAK_ADMIN_PASSWORD=admin \
  keycloak-custom start-dev

Theme Structure

The theme follows the standard Keycloak structure with custom styling applied to login pages.

Directory Layout 
custom/
└── login/
    ├── theme.properties
    └── resources/
        ├── css/
        │   └── styles.css
        └── img/
            └── favicon.svg

Configuration

Theme Properties

The theme.properties file defines the theme configuration:

parent=keycloak
import=common/keycloak

# Custom styles - listed last to properly override parent styles
styles=css/styles.css

Custom Favicon

To add a custom favicon, place your favicon.ico in custom/login/resources/img/ and override the template.ftl file to include:

<link rel="icon" href="${url.resourcesPath}/img/favicon.ico" />

Note: The favIcon property in theme.properties only works for admin and account consoles, not login pages. See this discussion for more details.

Customization

Modify custom/login/resources/css/styles.css to adjust colors, fonts, and layout. The theme inherits from the base Keycloak theme, so you only need to override specific properties.

Theme Activation

1

Access Admin Console

Navigate to your Keycloak admin console at http://localhost:8080

2

Select Realm

Choose the realm where you want to apply the theme from the dropdown menu.

3

Configure Theme

Go to Realm Settings → Themes tab

4

Apply Custom Theme

Select custom from the Login theme dropdown and save.

Requirements

Keycloak Version

Tested with Keycloak 26.1.5. Should work with versions 20.x and above.

Docker

Docker engine 20.10 or higher recommended for building and running the container.
